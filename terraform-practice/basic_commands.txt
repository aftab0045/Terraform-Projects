1.terraform init
 Initializes Terraform project by downloading providers and setting up backend.

2.terraform plan
 Shows execution plan (what Terraform will create/update/delete) without making changes.

3.terraform apply
 Applies the configuration and creates/updates infrastructure.

4.terraform destroy
 Deletes all resources managed by Terraform.

5.terraform state list
 Displays all resources currently tracked in Terraform state file.

6.terraform state show <resource>
 Shows detailed information about a specific resource from state file.

7.terraform state mv <source> <destination>
 Moves/renames a resource in the state file without recreating it.

8.terraform state rm <resource> (not terraform rm ❌)
 Removes a resource from state file without deleting actual infrastructure.

9.terraform validate
 Checks whether Terraform configuration syntax is valid.

10.terraform fmt
 Formats Terraform code into standard readable structure.

11.terraform taint <resource>
 Marks a resource to be destroyed and recreated in next apply.

12.terraform import <resource> <id>
 Imports existing infrastructure into Terraform state.

13.terraform apply --auto-approve
Applies changes without asking for manual approval.
